## ⚠️ 常见问题排查

### 问题1: 控制台显示 "i18n not initialized"

**原因:** i18n.js 未正确加载

**解决方法:**
1. 检查 Network 标签是否有 `/static/js/i18n.js` 请求
2. 确认状态码为 200
3. 检查 HTML 第1152行是否有: `<script src="/static/js/i18n.js"></script>`

---

### 问题2: 点击国旗按钮无反应

**原因:** toggleLanguage 函数未定义或window.i18n为undefined

**排查步骤:**
在Console中执行:
```javascript
typeof window.toggleLanguage  // 应该返回 "function"
typeof window.i18n            // 应该返回 "object"
```

如果返回 "undefined",检查:
- HTML 是否正确引入了 i18n.js
- JavaScript 代码是否有语法错误

---

### 问题3: AI返回的内容仍是中文(即使在英文模式下)

**原因:** language参数未正确传递到后端

**排查步骤:**
1. 检查 Network 标签中 `/api/generate_recipe` 请求的 Request Payload
2. 确认是否包含 `"language":"en-US"`
3. 如果没有,检查 index.html 第2139行是否有: `language: currentLanguage`

---

### 问题4: 语言文件404错误

**原因:** 后端缺少locales路由

**解决方法:**
确认 food_guardian_ai_2.py 第1077-1085行存在:
```python
@app.route('/locales/<lang>.json')
def get_locale_file(lang):
    """提供语言文件"""
    from flask import send_from_directory
    try:
        return send_from_directory('locales', f'{lang}.json')
    except:
        return send_from_directory('locales', 'zh-CN.json')
```

---

### 问题5: 刷新页面后语言重置为中文

**原因:** localStorage未正确读取

**排查步骤:**
在Console中执行:
```javascript
localStorage.getItem('fgai_language')
```

如果返回 null,说明:
- i18n.js 中的 init() 方法可能有问题
- 或者浏览器禁用了localStorage
