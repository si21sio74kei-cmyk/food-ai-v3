# 食谱生成计数器功能说明

## 🎯 修复内容

### 问题描述
1. **自动评估逻辑错误**：每次生成食谱都会立即进行营养评估，而不是等待3次后才评估
2. **缺少计数器显示**：用户不知道当前是第几次生成
3. **无法重置进度**：没有一键还原功能

### 解决方案

#### 1. ✅ 添加可视化计数器
在"生成环保菜单"按钮上方添加了绿色卡片，显示：
- 📊 **当前生成次数**（大字体显示）
- **进度条**（0/3 → 1/3 → 2/3 → 3/3）
- **提示文字**："每生成3次后自动进行营养评估"

#### 2. ✅ 添加一键还原按钮
在"生成环保菜单"按钮旁边添加了橙色按钮：
- 🔄 **一键还原**：点击后重置计数器为0
- 带有确认对话框，防止误操作
- 重置后会显示Toast提示

#### 3. ✅ 修复计数器逻辑

**修复前的问题**：
```javascript
// ❌ 错误逻辑：每次调用都+1，即使API失败也会增加
const generationCount = (latestAppData.generation_count || 0) + 1;
```

**修复后的逻辑**：
```javascript
// ✅ 正确逻辑：循环计数（0→1→2→3→1→2→3...）
const currentCount = latestAppData.generation_count || 0;

if (currentCount >= 3) {
    // 已经达到3次，重置为1（新的循环开始）
    latestAppData.generation_count = 1;
} else {
    // 未达到3次，正常递增
    latestAppData.generation_count = currentCount + 1;
}

const generationCount = latestAppData.generation_count;
```

**关键改进**：
1. **循环计数**：达到3次后自动回到1，开始新的循环
2. **失败回滚**：如果API调用失败，计数器会回滚（-1）
3. **实时UI更新**：每次计数变化都立即更新界面显示

## ⚠️ 注意事项

1. **计数器持久化**
   - 计数器保存在 `fgai_local_data.json` 文件中
   - 即使关闭浏览器再打开，计数器仍然保留
   - 只有点击"一键还原"才会重置

2. **循环机制**
   - 达到3次后会自动回到1，而不是继续增加到4、5、6...
   - 这是为了持续监控用户的饮食习惯

3. **失败保护**
   - 如果API调用失败，计数器会回滚
   - 确保计数准确反映成功的生成次数

4. **多设备同步**
   - 如果在多个设备上使用，每个设备的计数器是独立的
   - 因为它们使用各自的本地数据文件
