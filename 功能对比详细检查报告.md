## 🎯 页面切换逻辑对比

### 原版 (Tkinter)
```python
# 顶部导航栏
- 左侧: 标题 "FoodGuardian AI"
- 右侧: [📸 拍照识菜] [🎤 语音交互] [☰ 菜单▼]

# 菜单下拉
点击"☰ 菜单" → 显示tk.Menu下拉菜单
  ├─ 首页
  ├─ 智能食谱
  ├─ 营养分析
  ├─ AI 助手
  ├─ 智能冰箱
  ├─ 采购清单
  └─ 关于项目

# 页面切换
self.show_page('home')  # 隐藏其他页面,显示目标页面
```

### Web版 (iOS风格)
```html
<!-- 顶部导航栏 -->
<div class="top-bar">
  <div class="logo">🍽️ FoodGuardian AI</div>
  <div class="top-actions">
    <button onclick="openImageRecognition()">📸</button>
    <button onclick="openVoiceInteraction()">🎤</button>
  </div>
</div>

<!-- 底部TabBar -->
<div class="tab-bar">
  <div onclick="switchPage('home')">🏠 首页</div>
  <div onclick="switchPage('recipes')">🍳 智能食谱</div>
  <div onclick="switchPage('nutrition')">🥗 营养分析</div>
  <div onclick="switchPage('chat')">🤖 AI助手</div>
  <div onclick="switchPage('fridge')">🧊 智能冰箱</div>
  <div onclick="switchPage('shopping')">🛒 采购清单</div>
  <div onclick="switchPage('about')">ℹ️ 关于</div>
</div>

<!-- 页面切换 -->
function switchPage(pageName) {
  // 隐藏所有页面
  document.querySelectorAll('.page').forEach(p => p.classList.remove('active'));
  // 显示目标页面
  document.getElementById('page-' + pageName).classList.add('active');
  // 更新TabBar激活状态
  document.querySelectorAll('.tab-item').forEach(t => t.classList.remove('active'));
  event.currentTarget.classList.add('active');
}
```

**对比结论**:
- ✅ 两者都能切换到7个页面
- ✅ 拍照/语音功能都在顶部,位置一致
- ✅ Web版TabBar更直观,无需多次点击
- ✅ Web版有视觉反馈(active状态)
- **优势**: Web版 > 原版 (用户体验更好)
