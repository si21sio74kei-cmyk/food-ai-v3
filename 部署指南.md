## 🌐 方案一：Vercel 部署（推荐）

### 步骤 1：上传代码到 GitHub

```bash
# 进入项目目录
cd "d:\MyDesktop\Food AI"

# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "准备部署到 Vercel"

# 关联 GitHub 仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/Food-AI.git

# 推送
git push -u origin main
```

### 步骤 2：在 Vercel 上部署

1. **访问 Vercel**
   - 打开 https://vercel.com
   - 使用 GitHub 账号登录

2. **导入项目**
   - 点击 "Add New..." → "Project"
   - 找到你的 "Food-AI" 仓库
   - 点击 "Import"

3. **配置项目**
   - **Framework Preset**: 选择 `Other`
   - **Root Directory**: 保持默认 `./`
   - **Build Command**: 留空
   - **Output Directory**: 留空
   - **Install Command**: `pip install -r requirements.txt`

4. **配置环境变量（重要！）**
   - 点击 "Environment Variables"
   - 添加以下变量：
     ```
     ZHIPU_API_KEY = 你的智谱API Key
     ZHIPU_API_KEY_TEXT = 你的智谱API Key Text
     ```
   - 点击 "Add"

5. **部署**
   - 点击 "Deploy"
   - 等待 1-2 分钟
   - 部署完成后，你会得到一个 URL，例如：`https://food-ai.vercel.app`

### 步骤 3：验证部署

访问你的 Vercel URL，测试以下功能：
- ✅ 首页加载
- ✅ 智能食谱生成
- ✅ 营养评估
- ✅ AI 对话

### Vercel 优势

- ✅ 完全免费
- ✅ 自动 HTTPS
- ✅ 全球 CDN 加速
- ✅ 自动部署（推送代码即部署）
- ✅ 自定义域名支持
- ✅ 免费 SSL 证书

### Vercel 注意事项

⚠️ **Serverless 函数限制（免费版）：**
- 执行时间：最多 10 秒
- 内存：1024 MB
- 负载大小：4.5 MB

💡 **如果 AI 调用超时：**
- Vercel 免费版函数超时时间为 10 秒
- 如果 AI 响应慢，可能触发超时
- 解决方案：升级到 Pro 计划（$20/月，超时时间 60 秒）

## ☁️ 方案二：Render 部署（备选）

### 为什么选择 Render？

- ✅ 完全免费
- ✅ 无执行时间限制
- ✅ 适合长时间运行的任务
- ⚠️ 免费实例会在闲置时休眠

### 部署步骤

1. **访问 Render**
   - 打开 https://render.com
   - 使用 GitHub 账号注册/登录

2. **创建 Web Service**
   - 点击 "New" → "Web Service"
   - 连接你的 GitHub 仓库
   - 选择 "Food-AI" 仓库

3. **配置服务**
   - **Name**: food-ai
   - **Region**: 选择最近的（如 Singapore）
   - **Branch**: main
   - **Root Directory**: 留空
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python food_guardian_ai_2.py`

4. **配置环境变量**
   - 点击 "Environment"
   - 添加：
     ```
     ZHIPU_API_KEY = 你的智谱API Key
     ZHIPU_API_KEY_TEXT = 你的智谱API Key Text
     ```

5. **部署**
   - 点击 "Create Web Service"
   - 等待部署完成
   - 获取你的 URL：`https://food-ai.onrender.com`

## 🐧 方案三：PythonAnywhere（简单）

### 适合场景

- ✅ 学习和测试
- ✅ 个人项目
- ⚠️ 免费版功能有限

### 部署步骤

1. **注册账号**
   - 访问 https://www.pythonanywhere.com
   - 注册免费账号

2. **上传代码**
   - 在 Dashboard 点击 "Files"
   - 上传整个项目文件夹
   - 或者使用 Git：
     ```bash
     git clone https://github.com/你的用户名/Food-AI.git
     ```

3. **配置 Web App**
   - 点击 "Web" → "Add a new web app"
   - 选择 "Flask"
   - Python 版本选择 3.10
   - 路径配置：
     - Source code: `/home/你的用户名/Food-AI`
     - Working directory: `/home/你的用户名/Food-AI`
     - WSGI configuration file: 编辑为导入 Flask app

4. **安装依赖**
   - 打开 Bash console
   - 运行：`pip install -r requirements.txt --user`

5. **配置 WSGI**
   - 编辑 WSGI 文件：
     ```python
     import sys
     path = '/home/你的用户名/Food-AI'
     if path not in sys.path:
         sys.path.append(path)
     
     from food_guardian_ai_2 import app as application
     ```

6. **重启应用**
   - 点击 "Reload" 按钮

## 🔧 常见问题解决

### 1. Vercel 部署后 API 调用失败

**问题：** 智谱 API 返回 401 错误

**解决：**
- 检查 Vercel 环境变量是否正确配置
- 确保 API Key 没有过期
- 在 Vercel Dashboard → Settings → Environment Variables 中检查

### 2. 静态文件（CSS/JS）加载失败

**问题：** 页面样式不正常

**解决：**
- 确保 `static` 和 `templates` 文件夹在仓库根目录
- 检查 Flask 的 `static_folder` 和 `template_folder` 配置
- Vercel 可能需要额外配置静态文件路由

### 3. 数据持久化问题

**问题：** 重启后数据丢失

**说明：**
- Vercel 是 Serverless 架构，不保存文件状态
- `fgai_local_data.json` 的数据会在每次部署后重置
- 解决方案：
  - 使用外部数据库（如 Supabase、MongoDB Atlas）
  - 或使用 Vercel KV Storage（付费）

### 4. 本地测试正常，部署后报错

**检查清单：**
- ✅ 所有依赖都在 `requirements.txt` 中
- ✅ 环境变量已配置
- ✅ 文件路径使用相对路径
- ✅ 没有使用绝对路径（如 `C:\Users\...`）
