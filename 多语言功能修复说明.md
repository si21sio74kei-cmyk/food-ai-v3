#### ✅ 添加locales路由

- **功能**: 提供语言文件访问接口 `/locales/<lang>.json`
- **降级策略**: 如果请求的语言文件不存在,自动返回中文

```python
@app.route('/locales/<lang>.json')
def get_locale_file(lang):
    """提供语言文件"""
    from flask import send_from_directory
    try:
        return send_from_directory('locales', f'{lang}.json')
    except:
        # 降级到中文
        return send_from_directory('locales', 'zh-CN.json')
```

## ⚠️ 注意事项

### 1. 部署环境要求
- **locales目录**: 确保`locales/`文件夹与`food_guardian_ai_2.py`在同一目录
- **static目录**: 确保`static/js/i18n.js`存在且可访问
- **文件权限**: 确保Web服务器有权限读取这些文件

### 2. 浏览器兼容性
- **localStorage**: 需要浏览器支持localStorage(现代浏览器都支持)
- **CustomEvent**: 需要支持CustomEvent API(IE11+)
- **Fetch API**: 需要支持fetch(现代浏览器都支持)

### 3. 已知限制
- **部分文本未国际化**: 目前只为核心UI添加了data-i18n属性,其他动态生成的内容(如AI返回的Markdown)需要手动处理
- **AI Prompt质量**: 英文Prompt是简化版,可能需要进一步优化以确保输出质量
- **图片识别/语音交互**: 这些功能的AI调用尚未添加多语言支持

### 4. 后续优化建议
1. **扩充翻译键**: 为所有页面的静态文本添加data-i18n属性
2. **完善英文Prompt**: 优化`build_recipe_prompt()`中的英文Prompt,确保与中文版语义等价
3. **其他API多语言化**: 为营养评估、AI对话等API添加language参数支持
4. **RTL支持**: 如果需要支持阿拉伯语等RTL语言,需要添加CSS方向支持
