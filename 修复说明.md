## 修复的问题

### 1. ✅ 营养标准文件加载失败
**问题描述：**
```
❌ 加载营养标准失败：[Errno 2] No such file or directory: 'un_nutrition_standards.json'
```

**原因：**
程序使用相对路径加载 `un_nutrition_standards.json`，当从不同目录启动程序时，无法找到该文件。

**解决方案：**
修改 `load_nutrition_standards()` 函数，使用脚本所在目录作为基准路径：
```python
script_dir = os.path.dirname(os.path.abspath(__file__))
json_path = os.path.join(script_dir, 'un_nutrition_standards.json')
```

**影响范围：**
- 所有需要加载营养标准的功能（营养评估、饮食推荐等）

### 2. ✅ 变量作用域错误 (NameError)
**问题描述：**
```
NameError: cannot access free variable 'e' where it is not associated with a value in enclosing scope
```

**原因：**
在异常处理的 `show_error()` 内部函数中直接引用外部变量 `e`，由于 Python 的闭包延迟绑定特性，导致变量访问失败。

**解决方案：**
在所有异常处理块中，先保存错误消息到局部变量，再在内部函数中使用：
```python
except Exception as e:
    error_msg = str(e)  # 先保存错误消息，避免作用域问题
    def show_error():
        # 使用 error_msg 而不是 str(e)
        self.recommendation_output.insert(tk.END, f"❌ 生成推荐失败：{error_msg}")
    self.after(0, show_error)
```

**修复位置：**
共修复 8 处相同的问题：
- 第 3443 行：生成饮食推荐
- 第 4136 行：为所有人群生成推荐
- 第 4931 行：生成菜单
- 第 5348 行：AI 生成环保食谱
- 第 5753 行：余料改造方案
- 第 5930 行：食材营养分析
- 第 6428 行：购物清单生成

### 3. ✅ 营养评估结果验证
**问题描述：**
当营养标准加载失败时，`nutrition_assessment()` 返回错误字典，但后续代码尝试将其当作正常评估结果处理，导致类型错误。

**解决方案：**
在 `check_and_auto_recommend()` 函数中添加评估结果验证：
```python
assessment = nutrition_assessment(total_intake, population_group)

# 检查评估结果是否有效
if not isinstance(assessment, dict) or 'error' in assessment:
    print(f"⚠️ 营养评估失败：{assessment.get('error', '未知错误')}")
    return
```

## 技术要点

### Python 闭包变量绑定
Python 的闭包使用**延迟绑定**（late binding），内部函数在**调用时**才查找外部变量的值，而不是在定义时。这会导致在循环或异常处理中出现意外行为。

**错误示例：**
```python
except Exception as e:
    def show_error():
        print(str(e))  # ❌ 可能出错
```

**正确做法：**
```python
except Exception as e:
    error_msg = str(e)  # ✅ 立即求值并保存
    def show_error():
        print(error_msg)  # ✅ 使用局部变量
```

### 文件路径最佳实践
始终使用绝对路径或基于脚本位置的相对路径，避免依赖工作目录：
```python
# ❌ 不推荐：依赖当前工作目录
with open('config.json', 'r') as f:

# ✅ 推荐：基于脚本位置
script_dir = os.path.dirname(os.path.abspath(__file__))
config_path = os.path.join(script_dir, 'config.json')
with open(config_path, 'r') as f:
```
