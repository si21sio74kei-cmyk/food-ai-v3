## 🔍 常见问题排查

### 问题1：显示"网络错误"

**可能原因及解决方案：**

#### 原因A：服务器未启动
```bash
# 检查服务器是否运行
# 看PowerShell窗口是否有以下输出：
* Running on http://127.0.0.1:5000
```

**解决：**
```bash
python food_guardian_ai_2.py
```

---

#### 原因B：API密钥无效
**检查服务器端日志：**
```
🤖 [AI调用] 开始调用智谱 API...
   🔄 尝试模型: GLM-4-Air
      ❌ HTTP错误: HTTP 401
   ❌ 所有模型调用失败: HTTP 401
```

**解决：**
1. 检查 `food_guardian_ai_2.py` 第24行的API密钥
2. 登录智谱AI官网验证密钥是否有效
3. 如果密钥过期，申请新密钥

---

#### 原因C：API超时
**检查服务器端日志：**
```
🤖 [AI调用] 开始调用智谱 API...
   🔄 尝试模型: GLM-4-Air
      ❌ 超时: glm-4-air 超时
   🔄 尝试模型: GLM-4-Flash
      ❌ 超时: glm-4-flash 超时
   ❌ 所有模型调用失败: glm-4-flash 超时
```

**解决：**
1. 检查网络连接
2. 增加超时时间（修改 `food_guardian_ai_2.py` 第26行）：
   ```python
   API_TIMEOUT = 120  # 从90改为120
   ```

---

#### 原因D：CORS跨域问题
**检查浏览器控制台：**
```
Access to fetch at 'http://localhost:5000/api/generate_recipe' from origin 'null' has been blocked by CORS policy
```

**解决：**
确保通过 `http://localhost:5000` 访问，而不是直接打开HTML文件

---

### 问题2：刷新后出现旧记录

**原因：** `fgai_local_data.json` 中有历史数据

**解决：**
```bash
# 运行清理脚本
清理数据并重新开始.bat
```

或手动删除：
```bash
del fgai_local_data.json
```

---

### 问题3：自动录入没有生效

**检查点：**
1. 浏览器控制台是否有错误？
2. Network标签中 `/api/save_intake` 请求是否成功？
3. 服务器端是否有日志输出？

**调试步骤：**
1. 打开浏览器控制台（F12）
2. 切换到Network标签
3. 生成食谱
4. 查看 `/api/save_intake` 请求：
   - 状态码应该是 200
   - Response应该包含 `{"success": true, ...}`
